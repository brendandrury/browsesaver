# 📚 Browsesaver

**Browsesaver** is a privacy-first, memory-efficient browser extension that archives complex web chat interfaces (Discord, LMArena) directly to your local filesystem in real-time.

Instead of relying on fragile API wrappers or heavy local databases, Browsesaver utilizes highly optimized DOM Mutation Observers to intercept, format, and batch messages as you scroll, exporting them seamlessly as incremental NDJSON files.

> **Note on Development:** This project was built utilizing AI.

## ✨ Features

- **Real-Time DOM Scraping:** Hooks into React-based Single Page Applications (SPAs) without causing main-thread UI lag.
- **Intelligent Markdown Extraction:** Recursively parses deeply nested HTML nodes to reconstruct clean Markdown, capturing bolding, italics, code blocks, and lists.
- **Streaming Text Stability:** Specifically handles streaming LLM outputs (like LMArena) by utilizing cryptographic-style content signatures and debouncing algorithms to ensure messages are only saved once they stabilize.
- **Memory-Efficient Spooling:** Batches intercepted messages in an ephemeral background script memory buffer, bypassing the need for heavy local storage limits.
- **Local-First Export:** Automatically offloads data to the local filesystem (`~/Downloads/Browsesaver/...`) via the browser's native Download API as incremental NDJSON files, ready for ingestion by downstream ML tools.

## 🛠️ Tech Stack

- **Extension API:** Manifest V3 (Chrome/Firefox compatible)
- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3
- **Downstream Processing:** Python (YAML configurations for data ingestion)

## 🏗️ How It Works

1. **Content Script (`content.js`):** Detects the active platform context (Discord channel, LMArena session). Watches the DOM for new message nodes. Extracts metadata (timestamps, authors) and parses HTML into clean Markdown.
2. **Background Script (`background.js`):** Acts as the central state manager. Receives batched messages, de-duplicates them based on content length/signatures, and queues them in a spool buffer.
3. **Native Download API:** Every few seconds, the spool flushes, converting the queue into `.ndjson` files and downloading them silently to the local filesystem for permanent storage and later processing.

## 🚀 Installation & Setup

### 1. Load the Browser Extension

- Open Chrome/Brave and navigate to `chrome://extensions/` (or `about:debugging` in Firefox).
- Enable **Developer Mode** in the top right corner.
- Click **Load unpacked** and select the root directory of this repository.

### 2. Configure the Downstream Parser

Browsesaver is designed to feed into a local parsing script (like a semantic search engine or local LLM pipeline). To configure where your data goes:

```bash
# Copy the example config
cp config.yaml.example config.yaml
```
